# 量化选股系统 - Web 仪表盘

## 概述

为 stock_picker_v2 量化选股系统打造的 Web 图形化控制台，基于 Flask + ECharts 构建，提供从选股结果到回测分析的全维度可视化。

## 文件结构

```
dashboard/
├── app.py                    # Flask 后端 (API + 路由)
├── start.py                  # 启动器 (自动检测依赖、打开浏览器)
├── gen_bat.py                # BAT 脚本生成器 (GBK 编码)
├── 启动仪表盘.bat            # Windows 启动脚本
├── templates/
│   └── index.html            # 单页面应用模板
└── static/
    ├── css/style.css         # 暗色金融风格样式
    └── js/dashboard.js        # 前端交互逻辑 + ECharts 图表
```

### 模拟交易持仓周期筛选

模拟交易数据按 3 天、5 天、10 天三种持仓周期并行模拟，同一笔交易会存储 3 条记录。页面顶部提供**持仓周期筛选下拉框**：
- **全部持仓周期**（默认）：显示所有记录
- **持仓 3 天 / 5 天 / 10 天**：仅显示对应周期的交易

### 个股综合分析

输入股票代码或名称，系统从四个维度进行综合分析：

| 维度 | 权重 | 分析内容 |
|------|------|----------|
| 基本面 | 35% | 营收增长率、净利润增长率、毛利率、ROE、资产负债率、EPS |
| 新闻面 | 20% | 近 30 天新闻情绪分析（正面/中性/负面关键词） |
| 资金面 | 35% | 主力净流入、北向持股（依赖 akshare，网络不通时优雅降级） |
| 技术面 | 10% | MA5/MA10/MA20、20日高低点、10日/20日涨幅 |

- **综合评分**：加权平均后给出 1-10 分评级和投资建议
- **合理价格区间**：基于支撑位、压力位和 PE 估值给出参考区间
- **涨跌颜色**：遵循 A 股习惯，涨红跌绿
- **免责声明**：所有分析仅供参考，不构成投资建议

## 数据源说明

仪表盘**不硬编码任何参数**，所有数据均从项目实际数据文件读取：

- **选股结果**：读取 `output/选股结果_YYYYMMDD_HHMMSS.json` 中最新文件
- **AI 预测**：`ai_predictions/YYYY-MM-DD.jsonl`（可单独查看，未合并到选股结果页）
- **回测报告**：`reports/回测报告_YYYYMMDD_HHMMSS.json`
- **因子权重**：`factor_weights.json`
- **模拟交易**：`trades/open_positions.parquet`、`trades/closed_trades.parquet`
- **系统配置**：`config.json`（API 脱敏后返回）

> 注意：飞书推送的内容可能来自不同运行阶段（如 9:21 竞价推送、盘中推送、盘后推送），不同阶段的选股结果可能不同。仪表盘显示的是 `output/` 目录下最新保存的选股结果文件。

## 常见问题

### 涨跌幅显示为 1111.00% 等夸张数字？

这是前端格式化错误。项目中的 涨跌幅 字段已经是百分比形式（如 `11.11` 表示 11.11%），原代码错误地乘以 100 导致显示为 1111.00%。已在 `dashboard.js` 中修复，使用 `formatChange()` 直接显示原值。

## 个股分析页面使用说明

### 默认列表模式
打开个股分析页面后，默认从系统最新因子得分文件 (`data/factor_scores/factors_*.parquet`) 读取 Top 500 股票列表，以表格形式展示。列表包含排名、代码、名称、综合得分、涨跌幅、换手率、量比、市值、PE、行业等字段。

- **点击任意行**：加载该股票的四维分析（基本面/新闻面/资金面/技术面）
- **搜索框实时筛选**：在列表中输入代码或名称，表格实时过滤
- **搜索任意 A 股**：输入不在列表中的代码或名称，点击"分析"按钮可通过 API 搜索全 A 股
- **返回列表**：在分析详情页点击"← 返回列表"按钮回到股票列表

### 新闻面跳转
新闻列表中的每条新闻标题均可点击，会在新标签页打开百度搜索该标题，方便查看新闻详情。
