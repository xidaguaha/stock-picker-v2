## 一、系统架构总览

```
stock_picker.py          主流程调度器（每一步都是一个 Step）
    │
    ├──> stock_fetcher.py       数据获取（实时行情、历史K线、财报）
    ├──> factor_calculator.py   因子计算（24个因子 → 综合得分）
    ├──> ml_scorer.py           【新增】LightGBM机器学习打分
    ├──> market_regime.py       【新增】市场环境自适应权重
    ├──> sentiment_data.py      【新增】舆情数据（akshare）
    ├──> llm_factor_miner.py    【新增】LLM因子挖掘
    ├──> stock_ai_analysis.py  AI外围信息分析（新闻+LLM评分）
    ├──> paper_trader.py        模拟交易（开仓/平仓/持仓管理）
    ├──> backtest_engine.py     回测引擎（TopN + 全市场IC + 分层回测 + 全样本验证）
    ├──> evolution.py           进化调度器（日/周/月三层迭代）
    │        │
    │        └──> factor_tuner.py     因子调优器（IC/ICIR分析 → 权重调整）
    │                 │
    │                 └──> evolution_record.py  迭代变更记录
    ├──> ai_feedback.py         AI评分反馈闭环
    ├──> data_accumulator.py    数据积累（全量行情+因子得分+概念映射）
    ├──> trading_calendar.py    【升级】动态交易日历（akshare+缓存+兜底）
    └──> logger.py              分层日志系统
```

**设计原则：**
- 新功能尽量外挂（新增 `.py`），不改动核心逻辑
- 旧代码用 `try/except` 可选导入，新模块缺失不影响主流程
- 配置文件统一用 `config.json`

## 二、各模块说明

### 2.1 stock_picker.py（主调度器）

**执行流程：**
```
Step1: 获取全市场股票池（剔除ST/停牌/北交所/亏损/高PE/小市值/低价）
Step2: 获取实时行情（东方财富首选 → 腾讯 → 新浪降级）+ 交叉验证
Step3: 获取概念板块（55+类关键词匹配 + 知名个股硬编码）
Step4: 计算24个因子 → 综合打分
  ├── 基础因子（动量/趋势/量能/估值/规模/技术/概念/质量/成长/机构/研报/景气）
  ├── 舆情因子（情绪_市场 + 情绪_板块，来自akshare）
  ├── LLM因子（LLM_信号，来自大模型市场模式分析）
  ├── 【ML打分】LightGBM模型（有模型时使用，无模型回退加权平均）
  ├── 【市场自适应】根据牛/熊/震荡动态调整权重
  └── 【行业分散】同行业最多2只
Step4.5: AI外围信息分析（对Top500调用LLM分析）
Step5: 选股（按综合得分排序，行业分散取TopN）
Step6: 推送到飞书（含AI分析结果字段）
Step7: 历史归档（CSV+JSON+元数据+溯源）
Step8: 触发进化（调用 evolution.py）
```

### 2.2 stock_ai_analysis.py（AI分析模块）

**三源新闻抓取：**
| 来源 | 接口 | 内容 |
|------|------|------|
| 新浪财经 | 个股新闻页 | 主力来源，零鉴权 |
| 东财公告 | np-anotice-stock | 公司公告/重大事项 |
| 东财研报 | reportapi.eastmoney | 机构研报/评级 |

**模型fallback链（全部免费）：**
```
deepseek-ai/DeepSeek-R1-0528-Qwen3-8B（主模型）
  → THUDM/GLM-Z1-9B-0414（9B推理）
  → THUDM/GLM-4-9B-0414（9B通用）
  → Qwen/Qwen2.5-7B-Instruct（7B通用）
  → Qwen/Qwen3-8B（兜底）
```

**缓存逻辑（按日期）：**
```
load_cache() 检查 ai_analysis_cache.json：
  有今天的数据 → 保留今天的，清除昨天的 → 返回（不重新分析）
  没有今天的数据 → 返回空（今天第一次，重新分析）

效果：今天不管打开多少次，只要跑过一次AI分析，后续都用缓存
     明天第一次打开时清掉今天的，重新抓取分析
```

### 2.3 ml_scorer.py（LightGBM ML打分）

**工作流程：**
```
历史因子得分(data/factor_scores/) + 涨跌幅
    ↓ 训练LightGBM回归模型（截面预测）
    ↓ 预测：因子得分 → 涨跌幅排名百分位(0-1)
    ↓ 归一化到0-100 → 综合得分
    ↓ 无模型时自动回退到加权平均
```

**模型存储：** `cache/lgb_model.pkl`（pickle格式，避免中文路径问题）

### 2.4 market_regime.py（市场环境自适应）

**判断指标：** 沪深300均线位置（MA20 vs MA60）、20日波动率、20日涨跌幅

**权重策略：**
| 环境 | 动量 | 估值 | 概念 | 质量 |
|------|------|------|------|------|
| 牛市 | 0.22↑ | 0.08↓ | 0.12↑ | 0.02↓ |
| 熊市 | 0.10↓ | 0.18↑ | 0.04↓ | 0.06↑ |
| 震荡 | 0.18 | 0.12 | 0.08 | 0.03 |

### 2.5 sentiment_data.py（舆情数据）

**数据源：** akshare市场情绪指数 + 概念板块热度排名
**新增因子：** 情绪_市场(0.02) + 情绪_板块(0.02)

### 2.6 llm_factor_miner.py（LLM因子挖掘）

**工作流程：** 热门概念 → LLM分析市场动态 → 结构化JSON信号 → 缓存24小时 → LLM_信号因子(0.02)

**信号维度：** 政策方向、资金流向、板块轮动、催化剂事件

### 2.7 backtest_engine.py（回测引擎）

| 函数 | 说明 |
|------|------|
| `run_backtest()` | TopN验证：1/3/5/10/20天收益 |
| `run_full_market_ic()` | 全市场IC分析 |
| `run_quantile_backtest()` | 分层回测（五分位单调性） |
| `run_full_sample_validation()` | 全样本验证 |

### 2.8 evolution.py + factor_tuner.py（进化系统）

| 层级 | 触发条件 | 动作 | 频率 |
|------|---------|------|------|
| 日监控 | 每次运行 | 快速IC监控，只预警不调整 | 每天 |
| 周迭代 | ≥5天 + ≥5交易日 | IC分析→权重微调（EMA α=0.12） | 约每周 |
| 月迭代 | ≥20天 + ≥15交易日 | 深度优化+因子淘汰（EMA α=0.30） | 约每月 |

### 2.9 trading_calendar.py（动态交易日历）

**数据源优先级：** 缓存(7天) → akshare动态获取 → 硬编码兜底

### 2.10 盘后回测迭代 run.py

**流程（不拉实时行情）：**
```
Step 1: 加载过去5天快照（rglob子目录）
Step 2: 加载今日快照（从history/，不拉实时行情）
Step 3: K线回溯验证（backtest_engine.verify_pick_performance）
Step 4: 更新持仓（K线收盘价）+ 今日快照开新仓
Step 5: 进化迭代（快照数据，df_quotes=None）
Step 6: 保存报告
Step 7: 缓存清理（10项，30天滚动）
```

## 三、因子体系（24个因子，总权重=1.0）

| 类别 | 因子 | 权重 |
|------|------|------|
| 动量 | 涨跌幅/5日涨幅 | 0.09/0.07 |
| 趋势 | 均线位置/RSI | 0.07/0.05 |
| 量能 | 换手率/量比 | 0.05/0.04 |
| 估值 | PE/PB | 0.07/0.03 |
| 技术 | MACD金叉 | 0.07 |
| 规模 | 小市值 | 0.02 |
| 概念 | 热度 | 0.05 |
| 质量 | 盈利 | 0.03 |
| 成长 | 净利润/营收同比 | 0.07/0.05 |
| 机构 | 北向/基金 | 0.04/0.04 |
| 研报 | 覆盖度 | 0.03 |
| 景气 | 行业 | 0.02 |
| 情绪 | 市场/板块 | 0.02/0.02 |
| LLM | 信号 | 0.02 |
| AI | ai_score | 0.04 |

## 四、配置文件关键字段（config.json）

```json
{
  "weights": { ... },              // 10个英文key，映射到24个中文因子
  "ai_analysis": {
    "enabled": true,
    "provider": "custom",
    "model": "deepseek-ai/DeepSeek-R1-0528-Qwen3-8B",
    "fallback_models": [...],      // 6个免费模型降级链
    "score_weight": 0.08,
    "max_concurrent": 15,
    "analyze_top_n": 500
  },
  "feishu_webhook": "...",
  "top_n": 3
}
```

**注意：** weights中英文key通过weight_map自动映射到中文因子，新增因子（情绪/LLM）自动分配默认权重0.02，weights总和会自动归一化。

## 五、常见故障排查

| 现象 | 原因 | 解决 |
|------|------|------|
| AI分析使用缓存 | 今天已跑过 | 正常行为，按日期隔离 |
| AI分析不执行 | ai_analysis.enabled=false | 改为true |
| ML打分回退到加权平均 | 无训练数据或模型未训练 | 运行ml_scorer.py训练 |
| 市场自适应不生效 | akshare网络问题 | 检查网络，回退到默认权重 |
| 盘后回测报错 | df_today为None | 已修复，用快照数据替代实时行情 |
| 节假日判断错误 | trading_calendar缓存过期 | 自动从akshare刷新（7天缓存） |

### v6.0.2 更新日志（2026-06-25）

**致命Bug修复：**
1. **BASE_DIR NameError**：`stock_picker.py` 第 56 行在 BASE_DIR 定义前引用，源码模式启动崩溃
2. **盘中补充跳过**：`scheduler.py` 10:30 和 14:00 共用同一 phase 名，下午阶段被永久跳过

4. **因子淘汰失效**：`factor_tuner.py` EMA + 最低权重保护 0.02 导致淘汰因子仍有权重

### v6.0.5 (2026-06-29) — 全量实际运行测试修复

**实际运行竞价量化推送发现的致命 bug**：
- `竞价量化推送/run.py` 第 25 行 `from logger import` 在第 44 行 `sys.path.insert` 之前执行，导致 `ModuleNotFoundError: No module named 'logger'`，程序启动即崩溃
- 修复：调换 import 顺序，先设 sys.path 再 import

**腾讯数据源失败的真正原因**（经实际代码审查发现）：
- K 线 API 返回的成交量字段是字符串 `'6130667.000'`（带小数点），代码用 `int(item[5])` 转换会报 `ValueError`
- 修复：改为 `int(float(item[5]))`
- 腾讯实时行情被 `len(df) > 500` 硬编码阈值误判为失败
- 修复：改为动态阈值 `min_count = max(100, len(stocks) // 2)`

**东方财富股票列表 API 变更**：
- `push2.eastmoney.com` 现限制每次只返回 100 条（无论 `pz` 参数）
- 修复：当东方财富返回 < 1000 只时自动降级到新浪获取全量

**得分全 0 的根本原因**：
- `market_regime.get_regime_weights()` 返回的权重 key 是类别名（`momentum`, `volume`），而 `df` 列名是具体因子名（`动量_涨跌幅`）
- 权重 key 和因子列名完全不匹配，加权求和循环匹配不到任何因子，得分保持 0.0
- 修复：检查匹配数 < 3 时自动回退到 `FACTOR_WEIGHTS` 手工权重

**PB 查询性能优化**：
- 从 Baostock 逐只查询 2000+ 只 PB（耗时 50+ 分钟）改为从腾讯行情直接获取 `f[44]` 市净率字段（0 秒）
- Baostock 仅在行情数据不足 30% 时作为降级方案，且限制 Top500

**其他修复**：
- `_emerging_concept_sync` 因下划线开头不被 `import *` 导入，改为显式导入
- `akshare.stock.stock_ths_em` 在 akshare 1.18+ 已移除，用 try/except 保护
- 盘前数据检查 AI 文件字段名 `successful` → `analyzed`
